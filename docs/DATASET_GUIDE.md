# 数据集来源与演示选择指南

> 快速参考：各数据集背景、指标对比，以及每个功能模块推荐用哪个数据集演示效果更好。

---

## 一、数据集来源

### Benchmark
| 项目 | 内容 |
|------|------|
| **性质** | 公开基准数据集（代谢组学标准测评数据） |
| **来源** | 7 个连续上机批次的原始 Excel 文件（Batch1_0108 ~ Batch7_0306），系统自研脚本合并 |
| **规模** | 1715 样本 × 1180 特征 × 7 批次 |
| **分组** | 多组浓度梯度实验（`P1_AA_xxxx` / `P1_FA_xxxx` / `P1_Bio_xxxx` / `P1_PP_xxxx` / `P1_SRM_xxxx` / `P1_Full_xxxx`），每组代表不同物质、不同浓度的阶梯实验 |
| **特点** | 特征为匿名离子（ionIdx），通过 m/z 精确质量匹配做注释；QC/Burn-in/Water/SRM 等多类控制样本齐全；7 批次跨度约 2 个月 |

### BioHeart
| 项目 | 内容 |
|------|------|
| **性质** | 公开数据集（心脏代谢组学研究，MetaboAnalyst 风格格式） |
| **来源** | 外部下载的标准格式 CSV（`bioheart.csv`），带 15 批次批次标签和 QC 样本 |
| **规模** | 1300 样本 × 53 代谢物 × 15 批次（原始 1359，去除 59 个异常样本） |
| **分组** | Group 0（正常对照）/ Group 1（心脏病患者）/ QC |
| **特点** | 特征名即代谢物名称（如 "1-methylhistamine"），直接通过 HMDB 名称匹配注释，覆盖率 92.5%（49/53）；15 个批次跨度较大，批次效应中等 |

### MI（心肌梗死）
| 项目 | 内容 |
|------|------|
| **性质** | **国内临床真实数据**（两家顶级心血管中心的多中心队列） |
| **来源 — Batch 0** | **北京安贞医院** 氧化脂质发现集（`安贞医院-氧化脂质发现集.xlsx`） |
| **来源 — Batch 1** | **北京大学第三医院** 验证集（`北医三院验证集数据.xlsx`） |
| **合并脚本** | 自研 `combine.py`，对代谢物名称归一化后取两院交集（14 个代谢物） |
| **规模** | 902 样本 × 14 代谢物 × 2 批次（原始 929，去除 27 个异常样本） |
| **分组** | MACE（主要不良心血管事件，148例）/ non-MACE（无事件对照，754例） |
| **特点** | 14 个氧化脂质标志物，HMDB 注释覆盖率 92.9%（13/14）；两院间批次效应极强（centroid separation = 55.17），校正后完全消除 |

### AMIDE
| 项目 | 内容 |
|------|------|
| **性质** | 公开数据集（非靶向代谢组学，AMIDE 格式） |
| **来源** | 外部下载，统一预处理后存为 `amide.csv` |
| **规模** | 598 样本 × 6461 匿名质谱特征 × 3 批次 |
| **分组** | Group 0 / Group 1 / QC |
| **特点** | 特征全为匿名质谱信号，**无代谢物名称**，无法做注释/富集/MetaKG；差异分析可运行但结果无生物学解释价值 |

## 二、各模块推荐数据集

> 图标说明：⭐ 最佳效果 / ✅ 正常支持 / ⚠️ 支持但效果一般 / ❌ 不支持

### KPI 统计卡

| 数据集 | 推荐度 | 说明 |
|--------|:------:|------|
| Benchmark | ⭐ | 1715 × 1180 × 7，规模最大，数字最有冲击力 |
| BioHeart | ✅ | 1300 × 53 × 15，批次数最多（15个） |
| MI | ✅ | 902 × 14 × 2，规模较小 |
| AMIDE | ✅ | 598 × 6461 × 3，特征数最多但无意义 |

**推荐：Benchmark**

---

### PCA 可视化（批次校正前后对比图）

| 数据集 | 推荐度 | 说明 |
|--------|:------:|------|
| Benchmark | ⭐ | 7 批次颜色丰富，校正前后对比视觉反差明显 |
| BioHeart | ✅ | 15 批次，颜色最多，但点较密集 |
| MI | ⭐ | 2 批次颜色极为清晰（红/蓝各占半），校正前两团分离极明显，校正后完全合并，视觉反差最强 |
| AMIDE | ✅ | 3 批次，效果一般 |

**推荐：Benchmark（颜色丰富）或 MI（对比最强烈，适合快速说明效果）**

## 三、综合演示策略建议

### 主线演示（占 80% 时间）→ 选 Benchmark

所有模块效果在 Benchmark 上均最佳，且有唯一的 Autoencoder 填充评估，适合作为完整功能演示主线。

### 切换演示一（展示系统泛化性）→ 切换到 BioHeart

- 重点说"同一套系统在公开心脏代谢组学数据集上同样有效"
- 批次校正指标（双指标均改善）比 Benchmark 更"干净"，可以重点说
- 15 个批次的 PCA 图颜色丰富

### 切换演示二（展示临床价值）→ 提及 MI 即可，不必切换

可口头说："我们还整合了来自**北京安贞医院和北医三院**的真实心肌梗死临床数据，两院间批次效应高达 55.17，经系统处理后完全消除，证明系统对真实多中心临床数据同样有效。"

### AMIDE → 不推荐主动展示

可在问到"为什么有些功能没有"时，切过去展示 AMIDE 的限制提示，顺便解释系统对无注释数据的优雅降级处理。

## 四、一句话总结各数据集

| 数据集 | 一句话定位 |
|--------|-----------|
| **Benchmark** | 功能最全、数据最大、指标最好看，主力演示数据集 |
| **BioHeart** | 公开心脏代谢组学数据，15 批次、双指标改善，展示泛化性的最佳配角 |
| **MI** | 安贞医院+北医三院国内临床真实数据，批次效应最强（55→0），强调临床价值时提 |
| **AMIDE** | 6461 匿名特征，注释/富集/MetaKG 均不支持，用于展示系统对边界情况的处理 |
