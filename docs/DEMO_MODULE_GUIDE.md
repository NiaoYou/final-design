# 系统演示模块说明手册

> 本文档面向演示答辩场景，用通俗语言解释系统各模块的作用。  
> 支持数据集：**Benchmark**（1715 样本 × 1180 特征 × 7 批次）/ **BioHeart**（53 代谢物）/ **MI**（14 代谢物）/ **AMIDE**（6461 匿名特征）

---

## 系统是干什么的？

这是一个**代谢组学数据处理与分析 Web 平台**。

简单说：科学家在做代谢组学实验时，会产生大量数据（类似一个超大的 Excel 表格）。这些数据来自不同时间、不同批次的实验，存在各种问题（数据缺失、不同批次间系统误差等）。本系统的作用就是帮助科研人员**自动化地清洗、分析、可视化这些数据**，找出有意义的生物学信息。

系统支持 **4 个真实数据集的切换展示**，可在结果页顶部的数据集选择器中一键切换。

---

## 页面导航

系统共有 5 个页面，演示时按以下顺序走即可：

| 页面地址 | 页面名称 | 一句话说明 |
|----------|----------|------------|
| `/` | 首页 | 系统整体介绍 + 功能亮点概览 |
| `/import` | 数据导入页 | 上传原始 Excel 数据文件 |
| `/config` | 参数配置页 | 选择分析参数、启动处理流程 |
| `/result` | 结果仪表盘 | **核心演示页**，展示所有分析结果，支持多数据集切换 |
| `/history` | 历史任务页 | 查看历史分析记录 |

## 数据集选择器

结果仪表盘顶部有一个**数据集选择器**，支持在以下四个数据集之间切换：

| 数据集 | 规模 | 特点 |
|--------|------|------|
| **Benchmark** | 1715 样本 × 1180 特征 × 7 批次 | 最完整，全功能支持，含 Autoencoder 评估 |
| **BioHeart** | 53 代谢物 | 心血管代谢组学，92.5% 注释率 |
| **MI** | 14 代谢物 | 心肌梗死相关代谢物，92.9% 注释率 |
| **AMIDE** | 6461 匿名质谱特征 | 匿名特征，差异分析有效，注释/富集/MetaKG 不适用 |

**切换数据集后，页面所有区块自动刷新**，展示对应数据集的分析结果。

## 核心演示页各模块说明（/result 页面从上到下）

---

### 模块 0：数据集选择器

**位置**：结果页最顶部

**展示内容**：四个数据集的选择按钮，显示各数据集的简要描述和状态

**用大白话解释**：  
点击不同按钮，整个结果页就会切换到对应数据集的分析结果。就像在同一套仪表盘上切换"查看哪个工厂的数据"。

---

### 模块 1：KPI 统计卡

**位置**：数据集选择器下方，4 个数字卡片

**展示内容**：
- 样本数、特征数、Batch 数、缺失率（各数据集不同）

**用大白话解释**：  
就像一个"数据概览仪表盘"，让人一眼看出这次分析的数据规模。切换数据集后数字会自动更新。

---

### 模块 2：PCA 可视化（主成分分析图）

**位置**：KPI 卡下方，四宫格图 + 柱状图

**展示内容**：
- 四张图：分别展示"批次校正前/后"× "按批次着色/按样本组着色"的散点图
- 右边柱状图：每个主成分解释了多少数据变化

**用大白话解释**：  
PCA 是一种把复杂的高维数据"压缩"成二维图的方法。

- **校正前**：不同批次的样本点散布在不同区域（批次误差明显）
- **校正后**：不同批次的点混在一起（批次误差被消除）

切换数据集后 PCA 图自动更换为对应数据集的结果。

---

### 模块 3：批次校正指标对比卡

**位置**：PCA 图下方

**展示内容**：校正前后两个核心指标的数值对比（各数据集均有）

**Benchmark 关键数字**：
- **batch centroid separation**：5.38 → ≈ 0（100% 消除）
- **Silhouette(batch_id)**：-0.146 → -0.034（混合改善）

**用大白话解释**：  
这两个数字是定量证明批次效应被消除的客观依据，不是主观感觉。

---

### 模块 4：缺失值填充评估（仅 Benchmark）

**位置**：批次校正指标下方（仅在 Benchmark 数据集下显示）

**展示内容**：4 种填充方法的 RMSE 误差对比箱线图

| 方法 | RMSE | 说明 |
|------|------|------|
| **Autoencoder（深度学习）** | **0.2249** | 紫色高亮，最优 |
| KNN（近邻填充） | 0.2980 | 次优 |
| Mean（均值填充） | 1.0011 | — |
| Median（中位数填充） | 1.0361 | — |

**用大白话解释**：  
RMSE 越小说明猜得越准。我们用"Mask-then-Impute"框架（先遮住已知值再猜，看猜得有多准）来客观评估。

结论：**基于 PyTorch 的 Autoencoder 深度学习模型误差最小，比 KNN 低 24.5%，比均值填充低 78%**。

---

### 模块 5：差异代谢物分析（火山图）

**位置**：填充评估下方（全数据集支持）

**展示内容**：
- 组别选择器（从数据集的分组中选两组进行比较）
- 交互式火山图：横轴是 log2（倍数变化），纵轴是统计显著性
- 显著差异代谢物列表

**用大白话解释**：  
火山图中：
- **右边的红点**：在实验组里含量更高的代谢物（上调）
- **左边的蓝点**：在实验组里含量更低的代谢物（下调）
- **灰点**：两组间差异不显著，可以忽略

**演示推荐（Benchmark）**：选择 `P1_AA_0001` vs `P1_AA_1024`，结果会找出 **538 个显著差异代谢物**。

火山图点可悬浮查看代谢物名称、m/z 值、以及可点击的 HMDB/KEGG 数据库链接。

---

### 模块 6：通路富集分析

**位置**：火山图下方（Benchmark / BioHeart / MI 支持；AMIDE 显示提示）

**展示内容**：
- 气泡图（横轴=富集因子，纵轴=通路名，点大小=命中代谢物数，颜色=统计显著性）
- 力导向网络图（蓝色节点=代谢物，橙色节点=KEGG通路，可以拖拽）
- 富集结果明细表

**用大白话解释**：  
找出了差异代谢物之后，我们想知道："这些代谢物集中参与了哪些生物学过程？"

方法：用统计检验（超几何分布检验）判断某条代谢通路中的代谢物"出现得是否比随机情况明显更多"。

**Benchmark 演示结果**：D-氨基酸代谢通路（map00470）最为显著（富集因子 0.968，q 值 0.0002）。

切换为 **BioHeart / MI** 数据集后，富集分析基于该数据集的代谢物 HMDB/KEGG ID 进行，覆盖率 92%+。

> AMIDE 数据集特征为匿名质谱信号，无代谢物名称，无法进行通路富集。

---

### 模块 7：MetaKG 知识图谱溯源

**位置**：通路富集分析下方（Benchmark / BioHeart / MI 支持；AMIDE 显示提示）

**展示内容**：
- 大型力导向网络图
- 节点类型过滤按钮（代谢物 / 通路 / 反应 / 酶 / 药物）
- 关系类型过滤（has_pathway / has_reaction / has_enzyme 等）
- 代谢物搜索 + 黄色高亮定位

**各数据集子图规模**：

| 数据集 | 种子代谢物 | 总节点 | 总边 |
|--------|-----------|--------|------|
| Benchmark | 977 | 7,866 | 14,173 |
| BioHeart | 49 | 6,498 | 11,927 |
| MI | 13 | 268 | 825 |

**用大白话解释**：  
这是系统中规模最大、视觉效果最震撼的模块。它整合了 KEGG、HMDB、SMPDB 三大生物数据库的知识，展示代谢物与生化反应、酶、代谢通路、药物等实体的关联。

切换数据集后，自动加载对应数据集的子图。

**演示技巧**：
1. 在搜索框输入 `Alanine`（Benchmark）或数据集中的某个代谢物名，对应节点变为黄色
2. 开启"仅展示代谢物直连节点"，图谱更清晰
3. 图可以拖拽和缩放

---

### 模块 8：特征注释表格

**位置**：知识图谱下方（Benchmark / BioHeart / MI 支持；AMIDE 显示提示）

**展示内容**：代谢特征的分页注释表格，支持关键词搜索

**各数据集差异**：

| 数据集 | 注释策略 | 特征数 | 覆盖率 | 表格列 |
|--------|---------|--------|--------|--------|
| Benchmark | m/z 精确质量匹配 | 1180 | **100%** | Ion号 + m/z + 代谢物名 + HMDB/KEGG 单链接 |
| BioHeart | 代谢物名精确匹配 | 53 | 92.5% | 代谢物名 + 分子式 + 多 HMDB ID + 多 KEGG ID |
| MI | 代谢物名精确匹配 | 14 | 92.9% | 同上 |

**用大白话解释**：  
仪器检测到的原始数据只有一个个"m/z 值"（质荷比）或者代谢物名称，需要对应到具体代谢物并链接权威数据库。切换数据集后，表格自动切换为对应数据集的注释内容。

---

### 模块 9：方法对比实验表格

**位置**：特征注释下方

**展示内容**：
- 方法评估指标对比表（Silhouette 和 batch centroid separation）
- PCA 对比图
- 单方法 PCA 数据（可选择查看）

**Benchmark（5方法）vs BioHeart/MI（2方法）**：

| 数据集 | 对比方法 |
|--------|---------|
| Benchmark | mean / median / knn / baseline / **combat** |
| BioHeart / MI | mean / baseline |

**用大白话解释**：  
这里做了一个严格的方法横向对比实验：对比了不同数据处理策略的效果。

- **baseline 和 ComBat**（带批次校正）：batch centroid separation ≈ 0（效果最好）
- **KNN / Mean / Median**（只做了填充没有校正）：batch centroid separation ≈ 5.4（批次效应仍然存在）

用客观数字证明"批次校正步骤是必不可少的"。

---

### 模块 10：文件下载

**位置**：页面最底部

**展示内容**：可下载的分析结果文件（批次校正矩阵、评估报告、PCA 图等）

**用大白话解释**：  
分析完了之后，科研人员需要把结果数据下载回去用于论文写作或后续分析。非 Benchmark 数据集通过通用下载路由提供文件下载。

## 其他页面说明

### 首页（/）

首页用 **9 张技术亮点卡（3×3 宫格布局）** 概览了整个系统的核心创新点，以及一个 5 步处理流程图（导入 → 预处理 → 填充 → 批次校正 → 分析）。  
适合演示开场时快速展示系统全貌。

---

### 数据导入页（/import）

用于上传原始 Excel 实验数据（支持多 sheet 格式），系统会自动解析数据结构。  

本系统支持每个 Excel 包含 4 个 sheet：样本注射信息、信号强度矩阵、离子信息、代谢物注释。7 个批次的数据按照共同的"离子索引（ionIdx）"自动对齐合并为一个大矩阵。

---

### 参数配置页（/config）

上传数据后，可在这里选择：
- 预处理方式（Z-score 标准化 / log 变换）
- 缺失值填充方法（Mean / Median / KNN / Autoencoder）
- 批次校正方法（Baseline / ComBat）

页面顶部有一个 **6 步进度条**，实时显示当前处理到哪一步了。

---

### 历史任务页（/history）

查看所有历史提交的任务记录，包括任务状态、创建时间、使用的参数等。数据持久化存储在 SQLite 数据库中，重启服务后不会丢失。

---

## 演示推荐顺序

1. **首页** → 快速介绍系统功能定位（约 1 分钟）
2. **结果仪表盘**（直接跳转）→ 默认 Benchmark 数据集，按模块逐一介绍（约 8-10 分钟）
   - KPI 卡 → PCA 图 → 批次校正指标 → 填充评估（Autoencoder 最优）→ **火山图**（选组 + 点击运行）→ **通路富集**（点击运行 + 切换网络图）→ **MetaKG**（搜索高亮演示）→ 注释表格 → 方法对比
3. **切换数据集** → 选择 BioHeart，展示同样的功能在真实临床代谢组学数据集上的效果（约 2 分钟）
4. **导入页** → 简单展示上传界面（可选）
5. **历史页** → 展示任务记录（约 30 秒）

| 亮点 | 具体说明 |
|------|----------|
| 深度学习填充 | 基于 PyTorch 实现 Autoencoder，RMSE=0.2249，比 KNN 低 24.5% |
| 批次效应完全消除 | baseline/ComBat 双方法，batch centroid separation 从 5.38 降至 ≈ 0 |
| 100% 注释覆盖率 | Benchmark 1180 个特征全部通过 m/z 精确质量匹配成功注释 |
| 通路富集分析 | 超几何检验 + BH-FDR，发现 D-氨基酸代谢通路显著富集（q=0.0002） |
| 知识图谱 | 整合 KEGG/HMDB/SMPDB，7866 节点 × 14173 边，可交互探索 |
| 交互式火山图 | 538 个显著差异代谢物，悬浮显示注释信息，点击跳转数据库 |
| 5 方法横向对比 | 客观量化评估，证明批次校正的必要性 |
| **多数据集切换** | **4 个真实数据集（Benchmark/BioHeart/MI/AMIDE）一键切换，组件全面复用** |
